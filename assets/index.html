HTML

  • The head element containes information or metadata about the webpage
  • The body element represents the visible content shown to the user
  • HTML attributes [lang="en"] for example are always defined in the beginning element, never the closing element
  • The [title] element defines the title for a webpage
  • The [link] element connects an external resource to our html document
  • The [meta] element defines metadata like character set, description, keywords, author, and viewport
  • Elements that are within other elements are called CHILD ELEMENTS
  • Element and tag are used interchangeabl however element refers to everything from opening to closing tag where tag only refers to what is inside the brackets
  • [h1] [h2] [h3] and so forth represent the level of heading for a line of text
  • [p] for paragraph is the element used for exactly what it sounds like
  • [ul] [ol] and [li] are list elements, unordered, ordered, and list items, list items are what goes inside of the list
  • The [img] element containes information about images, the [src] attribute designates the location of the image
  • [br] creates a line of empty space
  • [a] or anchor creates a link to a webpage the [href] attribute containes the URL for the link
  • [section] elements contain content about a topic, the HTML CSS and JS notes are section elements

CSS

  • There are three ways of styling a webpage, internal CSS, external CSS, and inline CSS
  • inline CSS is used to style directly inline of the HTML file
  • internal CSS contains CSS rules within the head of an HTML file
  • external CSS uses the link element within the head to connect an external sheet of CSS rules
  • A selector defines the elements or attributes to which the declarations will apply
  • Declarations contain the property[color] and the value[blue]
  • The [margin] property adds space around the outside of an element
  • The [padding] property adds space around the inside of an element
  • The [class] attribute allows us to assign a class to specific elements in order to apply rules to given classes

Git

  • The [ls] command presents a list of files and directories on our computer
  • The [cd] command changes directories
  • The [pwd] command, short for print working directory, will print our current directory
  • The [cd ..] command will navigate backwards one directory
  • The [mkdir] command creates a directory
  • The [touch] command creates a file within a directory
  • The [git revert] command reverts to a specified version of our code
  • The [code .] command opens vs code
  • Use [git status] to check which branch we are on and the status of said branch
  • Use [git checkout] to move between branches and the main branch
  • Use [git checkout -b] to create a new branch
  • Use [git add -A] to stage all changes for the next commit
  • [git commit -m] commits all currently staged changes
  • [git pull origin main] pulls changes from the main branch in our github repo into our local environment
  • [git push origin] pushes our changes in our local environment to the github repo
  • Git Command Cheat Sheet

JavaScript

  • A [variable(var)] is a named container that stores data, the name must be unique
  • An assignment operator [=] is used to assign data to a variable [var shape = "circle";]
  • A [string] is a piece of or set of data that is represented in text and is between quotation marks
  • [control flow] is the order a computer reads code in a script, JavaScript reads from top to bottom
  • An [if statement] is a set of conditions that is run through until a condition is met
  • An [array] is a variable that stores a group of data typically that is related
  • An [index] is the location of a piece of data within an array, indices are squential and start with 0
  • A [for loop] is an iterative statement used to repeat code if conditions are met
  • Every line of JavaScript ends with a semicolon
